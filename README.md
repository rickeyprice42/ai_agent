# Personal AI Agent

Личный AI-агент с расширяемой архитектурой: память, инструменты, провайдер модели и CLI-интерфейс.

Сейчас в проекте уже есть:

- базовый цикл агента;
- постоянная память в JSON;
- реестр инструментов;
- mock-провайдер для локального запуска без API;
- готовая структура для подключения реальной модели позже.

## Структура

```text
AI_agent/
├─ .env.example
├─ main.py
├─ README.md
└─ src/
   └─ ai_agent/
      ├─ __init__.py
      ├─ agent.py
      ├─ config.py
      ├─ memory.py
      ├─ types.py
      ├─ llm/
      │  ├─ __init__.py
      │  ├─ base.py
      │  └─ mock_provider.py
      └─ tools/
         ├─ __init__.py
         ├─ base.py
         └─ builtin.py
```

## Быстрый старт

1. Создай виртуальное окружение при желании:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

2. Скопируй конфиг:

```powershell
Copy-Item .env.example .env
```

3. Запусти агента:

```powershell
python main.py
```

## Что умеет текущая версия

- вести диалог в консоли;
- запоминать заметки через фразы вида `запомни ...`;
- показывать, что хранится в памяти;
- вызывать инструмент времени;
- хранить память между перезапусками.

## Идея архитектуры

Мы разделили систему на независимые слои:

- `agent.py` управляет циклом рассуждений и вызовами инструментов;
- `llm/` отвечает за общение с моделью;
- `tools/` хранит расширяемые действия агента;
- `memory.py` сохраняет историю и заметки;
- `config.py` собирает настройки из окружения.

Это удобно тем, что позже можно заменить только один слой, не переписывая весь проект.
