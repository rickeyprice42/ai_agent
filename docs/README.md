# Структура проекта и анализ текущего состояния

Проект **AI_agent** содержит несколько основных каталогов и файлов:

- **`frontend/`** – фронтенд-приложение (React/Vite). Здесь находятся исходники UI, скомпилированные файлы (`dist/`), а также папка `node_modules`. Сейчас в HTML-шаблоне сайта указано название агента «Nova Agent».
- **`backend/`** – бэкенд на Python (FastAPI) с основными маршрутами `/api/bootstrap` и `/api/chat`, которые вызывают логику агента.
- **`src/ai_agent/`** – собственно код AI-агента (классы `Agent`, различные провайдеры LLM, инструменты и т.п.), который используется бэкендом через `agent_service`.
- **`docs/`** – документация (сюда будет добавлен итоговый roadmap).
- **`data/`** – хранилище данных (например, логи, модели или примеры диалогов), если есть.
- **`main.py`** в корне – точка входа для запуска `AgentApp` (возможно, CLI-приложение).
- **`requirements.txt`, `.env.example`, `.gitignore`** – служебные файлы (зависимости, пример env-переменных и т.д.).
- **`src-tauri/`** – конфигурация Tauri (пустая или заготовка).
- **`Nova AI-Agent/`** – отдельно созданный каталог со старым проектом (React + Tauri), который был создан ранее и сейчас устарел.  

**Выводы:** структура проекта смешанная. Есть отдельная заготовка Tauri-приложения (`Nova AI-Agent`), которую нужно удалить и сделать чистую интеграцию. Фронтенд и бэкенд разделены, и текущий веб-интерфейс использует React. В UI по умолчанию отображается «Nova», а новый агент должен называться «Avelin».

# UI/UX: текущий интерфейс и требования

Текущий UI (в React) оформлен в свето-бирюзовых тонах (судя по файлам стилей и сырцам). Требуется **полностью переделать дизайн в чёрно-белую стилистику**:

- **Цветовая гамма:** минимализм на контрасте белого фона и чёрных элементов. Можно добавить акцентный цвет (например, серый) для кнопок или ссылок. Это создаст строгий и лаконичный вид【15†L121-L130】【15†L163-L170】.
- **Тема оформления:** две темы – светлая и тёмная. В светлой теме фон белый, текст чёрный, а в тёмной – наоборот. Также нужно учесть дополнительный «печатающий» эффект у ответов агента (имитация последовательного ввода текста).
- **Логотип и имя агента:** поменять всюду название на **Avelin**. Добавить логотип в шапку/угол приложения и в окно (например, иконку окна). Логотип есть в двух вариантах (чёрный на белом фоне и белый на чёрном). 
- **Кнопки и шрифты:** использовать простые шрифты (например, без засечек), крупную кнопку отправки. Все элементы в чёрно-белых оттенках.  

Для примера: веб-сайты и приложения в монохромном стиле часто демонстрируют чистоту и фокус на контенте【15†L121-L130】【15†L163-L170】. Контраст чёрного и белого обеспечивает хорошую читаемость и строгий внешний вид. 

Кроме того, **эффект печати (typewriter effect)** для ответов AI сделает общение более живым. Например, можно в React-компоненте по символу добавлять буквы с задержкой через `setTimeout`【24†L142-L151】. Такой компонент (Typewriter) накапливает состояние строки и с помощью `useEffect` постепенно дописывает к ней символы текста, создавая иллюзию набора. Это придаст ощущение работы «нейросети» при выводе ответа. 

**Задачи:**

- Перекрасить UI в чёрно-белую гамму (менять стили или переменные CSS).  
- Заменить тексты **«Nova» → «Avelin»** в коде и HTML (например, в `<title>` и компонентах).  
- Вставить логотип: сделать так, чтобы при светлой теме отображался чёрный логотип, а при тёмной – белый. Логотип меняется вместе с темой (см. ниже).  
- Сделать анимацию печати для ответов агента (через компонент с `setTimeout`, как описано в примере【24†L142-L151】).  

# Реализация тем и переключателя

Необходимо три режима: **автоматическая смена** (в зависимости от времени суток), светлая и тёмная (ручная выборка). Подход:

- **JavaScript**: при загрузке приложения определять текущее время. Например, получить `const hour = new Date().getHours()`. Если час **>= 21** или **< 7**, устанавливать тёмную тему, иначе – светлую【12†L152-L158】. Это можно сделать в React-компоненте при инициализации (например, в корневом `App`). Для периодической актуализации можно запускать таймер на проверку каждую минуту (по аналогии с примером【12†L178-L186】).
- **CSS**: использовать CSS-переменные: задавать разметку в `:root` для светлой темы, и другую схему в классе `.dark-mode` или через `@media (prefers-color-scheme: dark)`. Когда JS определит тему, на корневой `<html>` ставится класс (например, `<html class="dark-mode">`), и CSS-переменные переключаются. В частности, логотип можно инвертировать или сменить изображение.  
- **Логотип**: примерно так: `<img id="logo" src="logo_black.png" />`. При тёмной теме в CSS либо меняем `filter: invert(1)` (как описано в примерe【17†L42-L49】), либо через JS меняем `src` на белый логотип. По возможности можно использовать CSS-фильтр `invert()`【17†L42-L49】, чтобы черный логотип стал белым в тёмной теме.
- **Переключатель темы**: добавить кнопку или переключатель, которым пользователь вручную включает «Светлая»/«Тёмная» тему. При клике переключать класс на `<html>` (аналогично стандартному toggler). *Важно:* автоматический режим основан на времени, но нужно дать ручной override или выбрать автопереключение. В настройках/меню UI можно реализовать выбор режима.

Таким образом, сочетаем **JS-логику по времени**【12†L152-L158】 и **CSS-фильтры для логотипа**【17†L42-L49】. В результате UI будет автоматически переходить в ночной режим в заданное время, а пользователь сможет переключаться вручную между светлой и тёмной темами.

# Десктоп-приложение на Tauri

Чтобы превратить веб-проект в настольное приложение, используем **Tauri** (Rust + веб-технологии). **Шаги реализации:**

1. **Удалить старую заготовку Tauri**. Удалите папку `Nova AI-Agent/`, поскольку она содержит устаревший проект Tauri (React+Rust). Убедитесь, что в корне остался лишь `frontend/` с актуальным UI и `backend/` с API. Также удалите (или переинициализируйте) папку `src-tauri/`, если там что-то есть.

2. **Установка Rust**. Tauri *требует* Rust-окружение【1†L485-L493】. Установите Rust через `rustup` (см. документацию Tauri):
   ```bash
   curl --proto '=https' --tlsv1.2 https://sh.rustup.rs -sSf | sh
   ```
   или через менеджер пакетов. Это позволит компилировать бинарники Tauri【1†L485-L493】. После установки убедитесь, что `rustc --version` выдаёт текущий Rust.

3. **Установка Tauri CLI**. Установите CLI-инструмент для Tauri. Например, если вы используете npm/pnpm:
   ```bash
   npm install -D @tauri-apps/cli
   ```
   Это зарегистрирует команду `tauri`. (Альтернативно можно установить через `cargo install tauri-cli`【6†L129-L137】.)

4. **Инициализация Tauri в проекте**. Перейдите в корень проекта (где находятся `frontend/` и `backend/`) и выполните:
   ```bash
   npx tauri init
   ```
   или `pnpm tauri init`, `cargo tauri init`【27†L479-L488】. Тулл забросит интерактивный мастер, где нужно ответить:
   - **App name** (например, `AvelinAgent`).
   - **Window title** (тоже «Avelin»).
   - **Assets directory**: укажите путь к собранному фронтенду (например, `../frontend/dist` или `../frontend/build`, в зависимости от сборки Vite). Это та папка, из которой Tauri берёт файлы UI при сборке.
   - **Dev server URL**: адрес локального сервера во время разработки (например, `http://localhost:3000`, если фронтенд запускается командой `npm run dev` на порту 3000)【27†L494-L503】.
   - **Build/Dev команды**: укажите соответствующие команды (`npm run build` и `npm run dev` для фронтенда).  

   После этого в корне появится папка `src-tauri/` с конфигурацией (`tauri.conf.json`)【27†L494-L503】.

5. **Перенос фронтенда**. Доработайте сборку фронтенда: выполняем `npm run build` (или аналог) чтобы сгенерировать финальный пакет `dist/`. Убедитесь, что путь указан правильно. В Tauri `tauri.conf.json` проверьте `build.distDir` на соответствие пути к вашим файлам.  

6. **Интеграция бэкенда**. По умолчанию Tauri запускает Rust-бинарник, а фронтенд загружает через WebView. Если ваш бэкенд на Python (FastAPI), нужно решить, как его запускать: можно, например, в скриптах Tauri перед стартом вызывать Python-сервер (через `@tauri-apps/command` или `invoke`), либо переписать API на Rust. Для простоты на этапе роадмапа можно оставить бэкенд как есть (требовать, чтобы при старте приложения запускался `uvicorn backend.main:app`). В любом случае в конечном билд-скрипте надо обеспечить, чтобы бэкенд работал локально и фронтенд мог к нему обращаться по `http://127.0.0.1:8000/api/...`.
7. **Тест и сборка**. Запустите приложение командой `tauri dev`【27†L518-L526】. Это запустит Dev-сервер фронтенда и откроет десктоп-окно. Если всё работает, выполняйте `tauri build` для получения релизного установщика/исполняемого файла.  

Справочные материалы по Tauri: официальный гид по интеграции описывает установку Rust и Tauri CLI【1†L485-L493】【6†L129-L137】 и показывает процесс `tauri init` в существующем проекте【27†L479-L488】【27†L494-L503】. Следуя им, вы настроите окружение для кроссплатформенного десктоп-приложения.

# Улучшение функционала

Помимо дизайна и упаковки, стоит продумать доводку самого AI-агента и инфраструктуры:

- **Логирование и отладка.** Добавить систему логов (логировать диалоги, ошибки) – это поможет быстро находить проблемы. Механизм логирования Python (например, `logging`) или сторонние мониторы.  
- **Автотесты.** Написать простые тесты для критичных функций (напр. обработка запросов или генерация ответов) поможет поддерживать качество кода.  
- **Документация к коду.** Комментировать функции и поддерживать актуальную документацию (файл `README.md`, раздел «Как разрабатывать»), чтобы в команде было проще ориентироваться.  
- **Улучшение AI.** Проанализировать логи диалогов, выявить слабые места (непонятные ответы, проблемы контекста). Возможно, подключить новую модель или доработать стратегию обработки. Например, расширить память агента, добавить больше «инструментов» (например, поиск по википедии, калькулятор и т.п.), если это предусмотрено.  
- **Переменные окружения.** Убедиться, что все ключи API и настройки вынесены в `.env` (есть `.env.example`). Настроить безопасную загрузку этих переменных.  
- **Интернационализация**. Если планируется мультиязычность (сейчас интерфейс на русском), можно сразу спланировать поддержку языков.  

Каждая такая доработка повысит устойчивость и удобство системы. Например, логирование облегчит диагностику, а тесты – отлов регрессий при дальнейших изменениях. Улучшенная модель и дополнительные инструменты сделают агента более полезным.
