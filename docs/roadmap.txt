# Roadmap Avelin

1. Анализ проекта и удаление лишнего
   - Удалить устаревший каталог `Nova AI-Agent/`.
   - Убрать старые и сгенерированные артефакты, которые не должны жить в репозитории.
   - Проверить актуальность `frontend/`, `backend/`, `src/ai_agent/`.
   - Убедиться, что фронтенд собирается, а backend импортируется и видит API-маршруты.

2. Обновление UI и названия агента
   - Заменить упоминания `Nova` на `Avelin`.
   - Подключить логотипы из `docs/light_logo.png` и `docs/dark_logo.png`.
   - Перевести интерфейс в строгую черно-белую палитру.
   - Добавить typewriter-эффект для ответов ассистента.

3. Реализация темы
   - Добавить режимы: auto, light, dark.
   - Переключать тему через класс на `html` и CSS-переменные.
   - В auto-режиме выбирать темную тему с 21:00 до 07:00.
   - Менять вариант логотипа вместе с темой.

4. [DONE] Проверка и расширение функционала
   - Добавлено базовое логирование backend-запросов и ошибок.
   - Проверены `/api/bootstrap`, `/api/chat`, `/api/health`.
   - Обновлена документация по структуре, запуску проекта, логам и быстрой API-проверке.

5. [DONE] Настройка окружения
   - Проверены Node.js/npm для frontend: Node.js 24.15.0, npm 11.12.1.
   - Проверены Python-зависимости для backend: Python 3.13.9, `pip check` без конфликтов.
   - Установлен Rust toolchain: `rustc 1.95.0`, `cargo 1.95.0`.
   - Добавлен Tauri CLI в `frontend`: `@tauri-apps/cli 2.10.1`, команда `npm run tauri`.
   - Проверены Windows-компоненты для Tauri: Visual Studio Build Tools 2022/MSVC и Microsoft Edge WebView2 Runtime.

6. [DONE] Продуктовая основа: пользователи, сессии и SQLite
   - Перенесено хранение памяти, истории диалогов, настроек и пользовательских данных в SQLite.
   - Спроектирована минимальная схема БД: `users`, `sessions`, `auth_accounts`, `chat_threads`, `messages`, `notes`, `model_settings`.
   - Добавлен слой доступа к данным в backend, чтобы агент не работал напрямую с JSON-файлами.
   - Реализован простой механизм инициализации SQLite-схемы при старте.
   - Сохранена совместимость с текущей локальной памятью: добавлен одноразовый импорт `data/memory.json`.

7. [DONE] Регистрация, вход и профиль пользователя
   - Добавлена обычная регистрация: email/логин, пароль, имя пользователя.
   - Добавлены вход и выход из аккаунта, хранение сессии и проверка текущего пользователя.
   - Подготовлен OAuth-контракт через Google.
   - Подготовлен OAuth-контракт через ВКонтакте.
   - Разделены данные пользователей: история, заметки и настройки модели привязаны к конкретному аккаунту.
   - Добавлен базовый профиль в UI: имя, логин, провайдер входа, действие выхода.

8. [DONE] Настройки AI-модели и provider-ов
   - Вынесен выбор provider-а и модели из кода в backend-настройки и пользовательские настройки.
   - Добавлен API для получения списка доступных provider-ов и моделей.
   - Добавлен API для смены активной модели без правки кода.
   - Поддержан текущий `mock` как безопасный fallback-режим.
   - Подготовлена конфигурация для Ollama: адрес сервера, имя модели, проверка доступности через ответ provider-а.
   - Добавлены UI-настройки модели: provider, модель, статус подключения, краткое описание режима.

9. [DONE] Agent Architecture: функциональная база
   - Выделен `ai_agent` как независимый модуль, который можно запускать из CLI, backend и будущего desktop-приложения.
   - Зафиксированы основные сущности: `Agent`, `Task`, `Tool`, `Memory`, `Planner`.
   - Агентное ядро не зависит напрямую от FastAPI и UI.
   - Backend оставлен тонким слоем API, который передает команды агенту и возвращает результат.
   - Результат: Avelin становится не чатом внутри FastAPI, а отдельной системой принятия решений.

10. Memory System: краткосрочная и долгосрочная память
   - Short-term memory: хранить контекст текущей сессии, ограничивать окно сообщений и готовить сжатие истории.
   - Long-term memory: хранить заметки пользователя, факты, результаты задач и важные решения.
   - Использовать SQLite как первый стабильный слой хранения.
   - Подготовить место для embeddings и семантического поиска на следующем этапе.
   - Retrieval: искать релевантные заметки/факты/результаты задач и подмешивать их в prompt.
   - Результат: агент помнит опыт и использует его при выполнении новых задач.

11. [IN PROGRESS] Tool System: агент начинает действовать
   - Развит единый `ToolRegistry` для всех инструментов.
   - Инструменты описываются через JSON schema: имя, описание, аргументы, результат.
   - Добавлено выполнение инструментов через LLM tool-calls с валидацией аргументов.
   - Добавлено безопасное выполнение и логирование вызовов.
   - Добавлен внутренний task manager.
   - Добавлен первый файловый инструмент: безопасное чтение текстовых файлов из рабочей папки инструментов.
   - Добавлен второй файловый инструмент: безопасная запись текстовых файлов в рабочую папку без перезаписи по умолчанию.
   - Далее нужны ограниченный shell и HTTP-запросы.
   - Результат: Avelin превращается из "болтуна" в исполнителя.

12. [IN PROGRESS] Task System: сердце Avelin
   - Введена сущность `Task`: id, статус, приоритет, описание, шаги выполнения, результат.
   - Начат жизненный цикл: create, plan, execute, complete, fail.
   - Добавлена очередь задач с последовательным выбором следующей задачи.
   - Добавлено хранение шагов и результатов выполнения.
   - Добавлены внутренние инструменты для создания, просмотра, планирования, запуска и обновления задач.
   - Результат: пользователь дает список задач, а агент ведет их до результата.

13. [IN PROGRESS] Planner / Reasoning
   - Добавлен первый планировщик, который разбивает задачу на шаги.
   - На каждом шаге выбирать подходящий инструмент.
   - Начат цикл `think -> act -> observe -> repeat`: агент умеет создать план, запустить следующую задачу и отметить текущий шаг.
   - Добавить контроль выполнения: обработка ошибок, retry, остановка при опасном действии.
   - Результат: агент перестает быть API-оберткой и начинает вести работу как исполнитель.

14. [IN PROGRESS] Context Orchestration
   - Добавлен `ContextBuilder`, который явно собирает prompt из текущей задачи, памяти и истории.
   - В prompt подмешиваются последние заметки пользователя и активная/следующая задача.
   - Ограничивать токены и приоритизировать контекст.
   - Отделить системные инструкции, память, план задачи и наблюдения от инструментов.
   - Подготовить summary для длинных сессий.
   - Результат: качество ответов и действий становится стабильнее на длинных задачах.

15. [IN PROGRESS] Safety Layer
   - Ввести ограничения на команды и файловые операции.
   - Добавлен sandbox для файловых инструментов и безопасная рабочая папка `TOOL_WORKSPACE_DIR`.
   - Добавлен лимит чтения/записи файлов и запрет записи за пределы рабочей папки.
   - Добавлен журнал действий агента: какой инструмент вызван, с какими аргументами, какой статус и результат получены.
   - Последние действия подмешиваются в prompt как наблюдения для следующих шагов.
   - Запрашивать подтверждение пользователя для потенциально опасных операций.
   - Результат: Ollama остается "мозгом", а Avelin становится контролируемой системой действий.

16. Стартовая страница и onboarding
   - Создать красивую стартовую страницу Avelin до входа в аккаунт.
   - Показать возможности агента: чат, память, заметки, локальная модель, смена темы, будущая desktop-версия.
   - Добавить формы регистрации и входа в едином стиле Avelin.
   - Добавить кнопки входа через Google и ВКонтакте.
   - После входа переводить пользователя в основной интерфейс чата.
   - Сохранить строгую черно-белую визуальную систему и адаптивность под desktop/mobile.

17. Создание desktop-приложения с Tauri
   - Инициализировать чистый Tauri-проект.
   - Настроить `tauri.conf.json` на `frontend/dist`.
   - Определить схему запуска FastAPI рядом с desktop-приложением.
   - Проверить, как работают SQLite, авторизация и настройки модели внутри desktop-сборки.
   - Проверить `tauri dev` и `tauri build`.

18. Полировка бренда и стартовой страницы
   - Починить логотипы: убрать белые края у черного варианта и черные края у белого варианта.
   - Проверить логотипы в light/dark темах, на стартовой странице, в sidebar и будущей desktop-иконке.
   - Добавить кнопку скачивания Avelin на первой странице сайта.
   - Пояснить на странице скачивания, что desktop-приложение устанавливается отдельно, а локальные модели нужно докачивать через Ollama или использовать облачный provider.

19. Desktop-установщик и дистрибуция
   - Настроить production-сборку Tauri и выпуск установщика для Windows.
   - Добавить имя приложения, иконки, publisher metadata и версию.
   - Проверить сценарий чистой установки, повторной установки и удаления.
   - Подготовить страницу загрузки установщика на сайте.
   - Добавить checksums/release notes для скачиваемых файлов.

20. Обновления desktop-приложения
   - Спроектировать механизм проверки обновлений.
   - Добавить состояние "доступно обновление" и кнопку обновления в UI.
   - Реализовать автоматическое обновление при следующем запуске или перезаходе, если пользователь согласился.
   - Проверить rollback/ошибки обновления и понятные сообщения для пользователя.

21. Приватная диагностика и телеметрия с согласием
   - Добавить экран согласия на диагностику при первом запуске и настройки opt-in/opt-out.
   - Отправлять только явно разрешенные данные: технические логи, версия приложения, сведения об ошибках, системные характеристики без лишней персональной информации.
   - Для базы данных и чатов сделать отдельное явное разрешение и режим ручной отправки диагностического пакета.
   - Добавить просмотр содержимого диагностического пакета перед отправкой.
   - Шифровать передачу, минимизировать данные и документировать, что именно собирается.
   - Не отправлять личные чаты, БД и данные компьютера скрыто или без явного согласия пользователя.

22. Хостинг сайта и серверная часть
   - Выбрать хостинг для сайта Avelin.
   - Настроить страницу скачивания desktop-приложения.
   - Добавить backend/API для приема разрешенной диагностики и логов.
   - Настроить хранение диагностических пакетов, ограничения доступа и удаление данных по запросу пользователя.
   - Подготовить домен, HTTPS, базовый мониторинг и резервные копии.

23. Тестовые инструменты агента для работы с файлами
   - Добавить инструмент открытия папки.
   - Добавить инструмент создания документа Word или совместимого `.docx`.
   - Добавить инструмент записи текста в документ.
   - Добавить базовое форматирование: заголовок, абзац, жирный/курсив, списки.

   - Ограничить инструменты безопасной рабочей папкой и показывать пользователю, какие файлы будут изменены.

24. Уведомления desktop-приложения

   - Добавить настройки уведомлений: включить/выключить, типы событий, quiet hours.

25. Завершение OAuth через Google и ВКонтакте

26. iPhone-приложение и remote-управление

   - Добавить безопасную авторизацию устройства и список доверенных устройств.

   - Продумать push-уведомления и ограничения, если ПК offline.
   - Подготовить отдельный roadmap для iOS-клиента после стабилизации desktop-версии.

27. Финальные улучшения и релиз

   - Проверить сценарии регистрации, входа, OAuth, смены модели, сохранения памяти и восстановления сессии.
